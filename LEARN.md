## 🔄 צעד אחר צעד: מה קורה כשמשתמש יוצר הזמנה

### דרך אפליקציית Next.js (חלק 2)

```
1.  המשתמש פותח http://localhost:3000/orders
2.  הדפדפן טוען את דף ההזמנות (קומפוננט לקוח)
3.  המשתמש ממלא: שם, מוצר, כמות
4.  המשתמש לוחץ "צור הזמנה"
5.  JavaScript קורא ל-fetch("/api/orders", { method: "POST", body: {...} })
6.  Next.js מנתב את זה ל-app/api/orders/route.js
7.  פונקציית POST רצה על השרת
8.  היא יוצרת הזמנה מדומה ומחזירה JSON
9.  הדפדפן מקבל את תגובת ה-JSON
10. React מעדכן את הדף להציג את התוצאה
```

### דרך מתזמר ה-Saga (חלק 4)

```
1. מריצים: npm run orchestrator
2. המתזמר שולח POST ל-localhost:4001/create-order
3. שירות הזמנות יוצר את ההזמנה → מגיב ✅
4. המתזמר שולח POST ל-localhost:4002/charge
5. שירות תשלומים מנסה לחייב...
   ├── הצלחה (70%) → מגיב ✅ → ממשיך לשלב 6
   └── כשל (30%) → מגיב ❌ → קופץ לגלגול אחורה
6. המתזמר שולח POST ל-localhost:4003/reserve
7. שירות מלאי שומר → מגיב ✅
8. 🎉 הכל הושלם! ההזמנה מוכנה.

גלגול אחורה (אם התשלום נכשל):
   1. המתזמר קורא ל-localhost:4001/cancel-order
   2. שירות הזמנות מבטל את ההזמנה
   3. 🔄 הכל חזר למצב שהיה לפני כן
```

## 🎭 מה זה תבנית ה-Saga?

### הבעיה
דמיינו שאתם קונים משהו באינטרנט. שלושה דברים צריכים לקרות:

1. **יצירת ההזמנה** במסד נתוני ההזמנות
2. **חיוב כרטיס האשראי** דרך מערכת התשלומים
3. **שמירת הפריט** במחסן

מה אם הכרטיס שלכם חויב, אבל הפריט אזל מהמלאי? 😱
צריך דרך **לבטל את החיוב** ו**לבטל את ההזמנה**.

### הפתרון: Saga
**Saga** הוא רצף של שלבים שבו לכל שלב יש **פעולה מפצה** (ביטול).

```
שלב                    פעולה מפצה (ביטול)
────                    ──────────────────
יצירת הזמנה      ←→     ביטול הזמנה
חיוב תשלום       ←→     החזרת תשלום
שמירת מלאי       ←→     שחרור מלאי
```

אם שלב כלשהו נכשל, ה-saga מריץ את הפעולות המפצות של כל השלבים **הקודמים**, בסדר **הפוך**.

### דוגמה מהעולם האמיתי
```
מסלול מוצלח (הכל עובד):
   יצירת הזמנה ✅ → חיוב תשלום ✅ → שמירת מלאי ✅ → הושלם! 🎉

מסלול כושל (התשלום נכשל):
   יצירת הזמנה ✅ → חיוב תשלום ❌ → ביטול הזמנה 🔄 → גולגל אחורה
```

## 🎵 מתזמר (Orchestrator) לעומת כוריאוגרפיה (Choreography)

יש שתי דרכים לתאם saga:

### 1. תבנית מתזמר (משמש בפרויקט הזה)

```
                  ┌──────────────┐
                  │   מתזמר      │  ← ה"בוס"
                  │  (מרכזי)     │
                  └──────┬───────┘
                    ┌────┼────┐
                    ▼    ▼    ▼
               ┌─────┐ ┌───┐ ┌─────┐
               │הזמנה│ │תשל│ │מלאי │
               └─────┘ └───┘ └─────┘

איך זה עובד:
- קומפוננט מרכזי אחד (המתזמר) שולט בהכל
- הוא אומר לכל שירות מה לעשות, אחד אחרי השני
- הוא מכיר את כל התהליך ומטפל בכשלים

יתרונות:                           חסרונות:
✅ קל להבנה                       ❌ נקודת כשל יחידה
✅ כל הלוגיקה במקום אחד           ❌ המתזמר יכול להיות מורכב
✅ קל לדיבוג                      ❌ תלות חזקה במתזמר
```

### 2. תבנית כוריאוגרפיה (החלופה)

```
               ┌─────┐    ┌───┐    ┌─────┐
               │הזמנה│───→│תשל│───→│מלאי │
               └──┬──┘    └─┬─┘    └──┬──┘
                  │         │         │
                  ▼         ▼         ▼
              [אירועים] [אירועים] [אירועים]

איך זה עובד:
- אין בקר מרכזי
- כל שירות מקשיב לאירועים ומגיב
- הזמנה נוצרה → שירות תשלומים שומע → מחייב
- תשלום בוצע → שירות מלאי שומע → שומר מלאי

יתרונות:                           חסרונות:
✅ אין נקודת כשל יחידה            ❌ קשה יותר להבין את הזרימה
✅ שירותים עצמאיים                 ❌ קשה יותר לדיבוג
✅ יותר סקיילבילי                  ❌ יכול להפוך ל"ספגטי אירועים"
```

### מה לבחור?

| קריטריון            | מתזמר | כוריאוגרפיה |
|----------------------|:-----:|:-----------:|
| פשטות               |  ✅   |     ❌      |
| דיבוג               |  ✅   |     ❌      |
| סקיילביליות         |  ❌   |     ✅      |
| עצמאות              |  ❌   |     ✅      |
| מעט שירותים (2-5)   |  ✅   |     ❌      |
| הרבה שירותים (10+)  |  ❌   |     ✅      |

**כלל אצבע**: תתחילו עם מתזמר. עברו לכוריאוגרפיה כשיש הרבה שירותים.

```
תורי הודעות (RabbitMQ, Kafka) בין שירותים
אימות, רישום, ניטור
מספר קבצים: routes, controllers, models, tests
CI/CD pipelines לפריסה אוטומטית
```

| היבט            | הפרויקט שלנו            | פרודקשן                       |

## 🚀 איך להריץ הכל

### 1. התקנת תלויות
```bash
npm install
```

### 2. הפעלת אפליקציית Next.js
```bash
npm run dev
```
פתח http://localhost:3000 בדפדפן.

### 3. הפעלת המיקרו-שירותים (פתח 3 טרמינלים נפרדים)
```bash
npm run order-service       # טרמינל 1 → פורט 4001
npm run payment-service     # טרמינל 2 → פורט 4002
npm run inventory-service   # טרמינל 3 → פורט 4003
```

### 4. הרצת מתזמר ה-Saga (פתח טרמינל רביעי)
```bash
npm run orchestrator
```

צפה בפלט הקונסול! הרץ מספר פעמים כדי לראות הן
הצלחה והן כשל (גלגול אחורה).
